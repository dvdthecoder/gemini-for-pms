# Module 1.1 — Welcome to Stride

## How This Course Works

This course has two layers:

**Layer 1 — Reference guides** (what you're reading now)
These pages explain the concepts. Read them before or after the interactive exercise.

**Layer 2 — Interactive lessons** (inside Gemini CLI)
Type `/start-1-1` inside Gemini CLI to launch the hands-on version of this lesson. Gemini becomes your coach and walks you through real exercises.

Both layers use the same fictional company: **Stride**.

---

## Your Situation at Stride

You just joined Stride as **Senior PM for Growth & Onboarding**. It's day one.

Your manager, Priya (CEO), drops a Slack message before you've even set up your laptop:

> *"Welcome! Quick context: activation is at 23%, should be 40%+. We're burning through trial users. Sam (data analyst) has the numbers. Would love a plan in 30 days. No pressure 😬"*

You have:
- A company overview doc: `company-context/overview.md`
- User personas: `company-context/personas.md`
- A product roadmap: `company-context/roadmap.md`
- Weekly user metrics: `company-context/data/user-metrics.csv`
- A feedback dump from churned users: `company-context/data/user-feedback.md`

You have Gemini CLI.

And you have 30 days.

---

## What Slash Commands Do

When you type `/start-1-1`, Gemini CLI reads the file `.gemini/commands/start-1-1.md` and runs it as a prompt. Gemini then coaches you through the exercise interactively.

Think of slash commands as lesson launchers — they start a structured conversation where you practice real PM work with company context already loaded.

**Available commands in this course:**

| Command | What it does |
|---------|-------------|
| `/start-1-1` | Meet Stride, learn how the course works |
| `/start-1-2` | Understand the 1M context window |
| `/start-1-3` | Your first real PM tasks |
| `/start-1-4` | Work with files using @ references |
| `/start-1-5` | Live web research with Google Search |
| `/start-1-6` | Project memory with GEMINI.md |
| `/start-2-1` | Write a PRD |
| `/start-2-2` | Analyze activation data |
| `/start-2-3` | Run competitive analysis |
| `/start-3-1` | Vibe coding — build a PM tool |

---

## Before You Start

Make sure you're running Gemini CLI from inside the `gemini-for-pms` course folder. The slash commands won't work properly if you run Gemini from a different directory.

To check, ask Gemini: *"What folder are we in?"*

If it's right, type `/start-1-1` to begin.
